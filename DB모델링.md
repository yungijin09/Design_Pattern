## ER 모델(ERD)

### ER 모델의 질적 특성

- 완전성(completeness) : 모든 업무 요구 사항을 표현
  
- 정확성(correctness) : 규칙에 맞게 정확히 작성
  
- 최소성(minimality) : 불필요한 중복 X
  
- 자명성(self-explanation) : 누구나 이해하기 쉽게 의미를 명확히 설명
  
- **확장성(extensibility)** : 업무 범위가 확대, 변경이 어렵지 않게 설계
  
- 표현성(expressiveness) : 요구 사항을 자연스럽게 표현, 표준 용어 사용
  
- 가독성(Readability) : 읽기 쉽게 표현

#### Entity

> **실존**하거나 **개념적인** 것
> 
> Java의 **Class**와 유사
> 
> 최소한 둘 이상의 **속성(column)** / **인스턴스(row)** 가 존재해야 함

- ##### 엔티티간의 관계
  
  - **관계 수** => 1:1, 1:n, n:n
    
    - 주문과 상품의 관계 생각!!!
      
      => n개의 상품이 주문될 수 있음 : 주문이 n개 된 것이 상품일 수 있음 == n:n
      
  - **선택성**
    
    - 점선이면 Optional
      
    - 실선이면 Mandatory
      
  - **식별자상속(PK 상속)**
    
    - Bar(=)가 없으면 식별자 상속, 없으면 비식별자 상속
- **관계 유형**
  
  - 기본 관계
    
  - 슈퍼타입 / 서브타입 관계
    
  - 재귀적 관계(Chain of Responsibility)
    
  - 병렬 관계
